Window服务注册表项的字段解析

驱动程序下 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services 的每个子项是服务程序或驱动程序的的配置项，无论是驱动还是符合服务控制协议的 Win32 程序。都通过这个项配置服务的一些基本信息，通过这个项，基本可以了解的信息如下：

1.驱动程序可以使用此子项来存储所需的数据

2.当 PnP 管理器使用其 DriverEntry() 函数初始化驱动程序时，它会将此项的注册表路径传递给函数。

3.Parameters 子项（可选）存储特定于驱动程序的数据。

4.Performance 子项 （可选）用于性能监视。

ImagePath

ImagePath 存储驱动程序映像文件的绝对路径。通常，该值类似于 %SystemRoot%\System32\drivers\driverName.sys 。但是，有时也会看到对以下内容的 svchost.exe 引用，例如：

%SystemRoot%\System32\svchost.exe -k netsvcs -p

此值由驱动程序 .inf 文件的强制 ServiceBinary 输入创建。

Start

“start”指定如何或何时启动服务：

  • 0：由引导加载程序加载（但未启动）。然后在内核初始化期间启动。
  • 1：在内核初始化期间，在 start 参数为 0 的服务之后启动。
  • 2：自动。由 smss.exe （会话管理器）或 services.exe （服务控制器）启动。
  • 3：手动。由服务控制管理器 （SCM） 启动。
  • 4: Disabled 4：已禁用
  • 5: Delayed start. 5：延迟启动。

起始值 1 表示这是内核设备或其他低级硬件设备驱动程序。对于适配器，将忽略起始值。在 msconfig.exe的“服务”选项页 下提供了一种方便的方法来禁止或保持服务的原始启动类型（以便以后再次启用）。

Type 类型

type 值指定服务的类型：

  • 0x01：内核设备驱动
  • 0x02：文件系统驱动程序（也是内核设备驱动程序）
  • 0x04：适配器的参数集
  • 0x10：可由服务控制器启动并遵守服务控制协议的 Win32 程序。
  • 0x20：可以与其他 Win32 服务共享进程的 Win32 服务。(暂未接触过)
  • 0x60：USER_SHARE_PROCESS模板（暂未接触过）
  • 0xE0：USER_SHARE_PROCESS实例（暂未接触过）

Other values

其他可见值包括：

  • BootFlags BootFlags（引导标志）
  • ConfigurationFlags 配置标志
  • DelayedAutostart 延迟自动启动
  • DependOnGroup 依赖组
  • DependOnService 依赖服务
  • Description 描述
  • DisplayName 显示名称
  • DriverPackageId
  • ErrorControl 错误控制
  • FailureActions FailureActions（失败操作）
  • FailureActionsOnNonCrashFailures
  • FailureCommand
  • Group 群
  • HasBootConfig
  • NdisMajorVersion
  • NdisMinorVersion
  • ObjectName (which seems to refer to an account such as NT AUTHORITY\NetworkService?)
    ObjectName（似乎指的是 NT AUTHORITY\NetworkService 等帐户？
  • Options 选项
  • PreshutdownTimeout
  • RebootMessage
  • RequiredPrivileges 必需权限
